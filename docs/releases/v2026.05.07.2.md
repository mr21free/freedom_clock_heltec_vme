# Freedom Clock 2026.05.07.2

## Highlights

- Fixed online firmware install failing with a connection error when the device followed the GitHub release asset redirect to the CDN.
- Added a live progress indicator to the setup page so the user sees what the device is doing during an online firmware install.

## Fixes

- Online firmware install now resolves the GitHub release asset redirect with a short probe request before downloading. This works around a known ESP32 HTTPClient limitation where cross-host HTTPS redirects (github.com → objects.githubusercontent.com) fail with a connection error on reconnect.
- Corrected the README description of the setup Wi-Fi password. It is stable per device and derived from the hardware device ID, not randomized on each boot.

## Improvements

- The setup page now shows a spinner and time-based step messages while an online firmware install is in progress: connecting to Wi-Fi, checking GitHub, downloading firmware, installing, and wrapping up. The Install button stays disabled throughout and the normal result message appears when the operation finishes.

## Builder Notes

- Build both public update packages before publishing:

```bash
./tools/FreedomClockSecurityTool.command build-manual-update --release-name freedom-clock-v2026.05.07.2 --board e213
./tools/FreedomClockSecurityTool.command build-manual-update --release-name freedom-clock-v2026.05.07.2 --board e290
```

- Use the `.bin` file matching the physical board model. Do not flash an E213 build onto E290 hardware, or the other way around.
