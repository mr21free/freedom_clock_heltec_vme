# Freedom Clock 2026.05.21.1

## Highlights

- Broader portfolio support: Freedom Clock now supports `USD`, `EUR`, and `CHF` across setup inputs, BTC price lookups, cached data, history, and device screens.
- More reliable offline behavior: online BTC price now falls back from CoinGecko to mempool.space, and last-good BTC price and balance are cached locally with a timestamp, so temporary Wi-Fi, market-data, or MQTT failures no longer collapse freedom calculations to misleading zero values.
- Better setup and update experience: the setup page has a more resilient save/update/copy flow, clearer privacy controls, grouped developer diagnostics, and improved firmware install progress.
- Hardware case files are now included under `hardware/case/`, including an editable OpenSCAD source file.

## New Features

- **Currency selector** — choose `USD`, `EUR`, or `CHF`. Static wealth, monthly expenses, online BTC price, MQTT price expectations, cached BTC price, settings display, and history stats all use the selected currency.
- **Last-good BTC cache** — successful BTC price and balance reads are stored locally in NVS with a timestamp and reused only when live data cannot be fetched.
- **Stale/offline indicators** — main, quote, settings, freedom change, and wealth change screens avoid false zero values and use `N/A`, loading states, and compact warning/no-Wi-Fi icons where appropriate.
- **Local time zones** — daily refresh uses a hardcoded POSIX time-zone list. Zurich / Central Europe is the default, and DST-aware zones adjust automatically.
- **Privacy screen toggles** — the setup page can hide the wealth-change and settings screens from normal FUNCTION-button navigation.
- **Developer diagnostics** — local developer builds can show copyable boot, setup portal, storage, battery, and history diagnostics from one Developer section.
- **3D printable case** — `hardware/case/` contains E290 case STL files and `freedom_clock_e290_case.scad`, which can be modified with OpenSCAD.

## Fixes

- **Setup page reliability** — Save, Check for Update, Install, Wi-Fi refresh, and Copy now have an early fallback controller for iOS captive browser cases where the larger setup script is delayed or not executed.
- **Copy feedback** — all setup-page copy buttons use a consistent `✓ Copied` / `Select + Copy` experience.
- **Firmware install flow** — software update uses the same overlay pattern as settings save and handles device restart/setup-mode edge cases more gracefully.
- **Button behavior** — `SETUP` (`BOOT / GPIO0`) opens setup with one press and factory resets when held for about 10 seconds. `FUNCTION` (`21 / GPIO21`) cycles through enabled screens with total press count: first enabled screen, second enabled screen, third enabled screen.
- **Unique setup Wi-Fi names** — setup SSIDs now use a short 6-character ID derived from the full hardware MAC instead of only the last 6 hex characters, avoiding the collision seen on multiple devices while keeping the name easy to type.
- **Factory reset and status screens** — reset, settings-saved, software-updated, and auto-update screens now share the same branded layout and dark/light theme behavior.
- **Quote layout and icons** — quote text wrapping, footer icon placement, heart/rocket icons, and separator positioning were tuned for the E290 layout.
- **Partial refresh artifacts** — manual screen refresh behavior was adjusted to reduce broken borders and noisy partial redraw regions on dark screens.

## Code And Release Hygiene

- The monolithic sketch was split into focused modules under `src/` for config, setup portal, display screens, OTA, BTC data, history, sleep scheduling, timezone support, diagnostics, and security helpers.
- `tools/check_release.sh` now runs whitespace checks and compiles both E213 and E290 firmware.
- The README now documents OPSEC tradeoffs for MQTT, online price lookup, setup PIN, developer diagnostics, and secure-device setup.

## Builder Notes

```bash
./tools/FreedomClockSecurityTool.command build-manual-update --release-name freedom-clock-v2026.05.21.1 --board e213
./tools/FreedomClockSecurityTool.command build-manual-update --release-name freedom-clock-v2026.05.21.1 --board e290
```

Use the `.bin` file matching the physical board model. Do not flash an E213 build onto E290 hardware, or the other way around.
