# Freedom Clock 2026.05.07.1

## Highlights

- Added Heltec Vision Master E290 support alongside the existing E213 build.
- Renamed the Arduino sketch to the shared `Freedom_Clock_HeltecVME.ino` entrypoint so both supported boards use one firmware source and one release line.
- Added model-specific firmware packages and updater selection for E213 and E290 release assets.
- Hardened online release checks, online firmware downloads, and CoinGecko price requests with HTTPS certificate validation.
- Randomized the setup Wi-Fi password each time setup mode starts instead of deriving it from the public setup SSID.

## New Functionality

- The firmware now detects the selected Heltec board profile at compile time and chooses the matching e-ink display driver and layout.
- The setup page now prefers model-specific release assets such as `FreedomClock-2026.05.07.1-E213-manual-update-open.bin` and `FreedomClock-2026.05.07.1-E290-manual-update-open.bin`.
- Legacy generic release asset names are still accepted as a fallback for older releases.
- Screen navigation now puts freedom change on one wake-button press, wealth change on two quick presses, and settings on three quick presses.
- Added a local Battery Calibration Log in setup mode so builders can copy recent voltage and percentage samples for tuning.
- Added E213 and E290 showcase photos plus a dedicated device-support guide.

## Fixes And Hardening

- Removed insecure TLS mode from online firmware update and price-fetch paths.
- Online update now fails closed if the device cannot sync time for certificate validation.
- The production helper now protects provisioning workspace, vault, per-device bundles, Secure Boot keys, flash-encryption keys, and local GitHub token files with private filesystem permissions.
- Expanded `.gitignore` coverage for secrets, private keys, build artifacts, firmware binaries, logs, local dev pages, and generated provisioning output.
- Updated release tooling so one release can build and publish separate E213 and E290 `.bin` assets.

## Builder Notes

- Build both public update packages before publishing:

```bash
./tools/FreedomClockSecurityTool.command build-manual-update --release-name freedom-clock-v2026.05.07.1 --board e213
./tools/FreedomClockSecurityTool.command build-manual-update --release-name freedom-clock-v2026.05.07.1 --board e290
```

- Use the `.bin` file matching the physical board model. Do not flash an E213 build onto E290 hardware, or the other way around.
- This release was verified on both E213 and E290 Arduino build profiles.
