# Freedom Clock 2026.05.08.1

## Highlights

- Setup page interactions now match the firmware install experience across all action buttons — spinners with live step messages instead of static text or page navigations.
- Added a developer flag to skip the first-boot welcome screen on unconfigured boot, preventing the device from sleeping on "Press any button" after a factory reset during development.

## Fixes

- Fixed the welcome screen appearing after an Arduino IDE firmware flash when the device had been factory-reset during testing. The root cause was that the NVS `welcome_seen` flag was cleared by the reset, causing the next unconfigured boot to show the welcome screen even though the developer expected to land in the setup portal. A new `SKIP_WELCOME_SCREEN` flag in `secrets.h` bypasses this for development builds.

## Improvements

- **Save button**: replaced the browser page-navigation to a confirmation screen with a full-screen overlay showing a spinner and "Saving settings…", then "Settings saved! Device is restarting…" once the device confirms. Validation errors surface back into the form without leaving the page.
- **Validate button** (renamed from "Test Connection"): shows an inline spinner with "Validating settings…" while the request is in flight, then reveals the result in the same spot. No more static "This can take a few seconds…" text.
- **Check for Update button**: shows an inline spinner with "Checking for updates…" while connecting and querying GitHub, then reveals the result inline.
- **Copy URL button**: replaced the distant status message above the release box with an in-place confirmation — the button text briefly changes to "✓ Copied" for 2.5 seconds, right where the user's eyes are.

## Developer Notes

- Add `#define SKIP_WELCOME_SCREEN 1` to your local `secrets.h` to always go straight to the setup portal on unconfigured boot. Never ship this flag to customers.

## Builder Notes

- Build both public update packages before publishing:

```bash
./tools/FreedomClockSecurityTool.command build-manual-update --release-name freedom-clock-v2026.05.08.1 --board e213
./tools/FreedomClockSecurityTool.command build-manual-update --release-name freedom-clock-v2026.05.08.1 --board e290
```

- Use the `.bin` file matching the physical board model. Do not flash an E213 build onto E290 hardware, or the other way around.
