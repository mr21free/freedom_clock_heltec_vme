# Freedom Clock 2026.05.30.1

## Fixes and reliability

- Fixed quote text wrapping on E290 so larger quote text no longer runs into the battery area or gets cut off at the right edge.
- Quote wrapping now derives its line width from the active display size, so E213 and E290 use the same layout logic with their own safe margins.
- Moved battery calibration log storage from NVS preferences to SPIFFS, with fallback loading from the previous NVS location. This keeps the large rolling battery log away from settings storage.
- Added battery-log storage status to the developer battery export, making it clearer whether samples loaded from SPIFFS, legacy NVS, or failed storage validation.

## Developer diagnostics

- Developer-only diagnostics now load on demand from `/developer-text` instead of being embedded directly into the setup page HTML.
- Added raw developer export links for boot path, app boot, setup portal, battery, storage, and history diagnostics.
- This keeps the setup page lighter and reduces the chance that large local diagnostic/history exports make the captive portal slow or fragile.

## Installation

- `FreedomClock-2026.05.30.1-E213-manual-update-open.bin` for normal E213 devices
- `FreedomClock-2026.05.30.1-E290-manual-update-open.bin` for normal E290 devices
- `FreedomClock-2026.05.30.1-E213-manual-update-secure.bin` for security-hardened E213 devices, if signed locally
- `FreedomClock-2026.05.30.1-E290-manual-update-secure.bin` for security-hardened E290 devices, if signed locally
