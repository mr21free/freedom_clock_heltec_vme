# Freedom Clock 2026.06.05.1

## Battery percentage calibration

- Updated the battery percentage curve using a full local discharge log from the real Freedom Clock battery and board.
- The estimate now follows measured runtime more closely instead of relying on a generic LiPo voltage curve.
- Mid and low battery readings are now less optimistic. For example, about `3.75V` now reads near one third remaining instead of half full.
- The bottom end now treats the last clean logged voltage, about `3.26V`, as effectively empty, while ignoring the final invalid reset sample.
- Battery percentage is now stabilized across deep-sleep wakes, so small voltage rebounds no longer make the displayed percentage tick upward during normal discharge. Larger voltage rises are still treated as charging/recharge events.
- After reset-like boots, the stabilizer can seed itself from the latest valid battery-log sample, avoiding a visible percentage jump when the device still has recent calibration history.

## Freedom and wealth change history

- Moved the daily wealth/BTC history store from NVS to SPIFFS, with fallback loading from the previous NVS location.
- This keeps the larger history record out of the small settings partition and makes the Freedom Change and Wealth Change screens more reliable over multiple days.
- Added history storage status to the developer history export so it is clear whether data loaded from SPIFFS, legacy NVS, or failed validation.
- Preserved history lifecycle diagnostics across normal settings saves, so `history_write_count` and the last-write day are no longer wiped by unrelated setting changes.
- Added a narrow repair path for devices that have only today's row but still have recent older write metadata, backfilling the missing recent days with the latest known value instead of leaving all change periods as `N/A`.

## Installation

- `FreedomClock-2026.06.05.1-E213-manual-update-open.bin` for normal E213 devices
- `FreedomClock-2026.06.05.1-E290-manual-update-open.bin` for normal E290 devices
- `FreedomClock-2026.06.05.1-E213-manual-update-secure.bin` for security-hardened E213 devices, if signed locally
- `FreedomClock-2026.06.05.1-E290-manual-update-secure.bin` for security-hardened E290 devices, if signed locally
