# Freedom Clock 2026.05.23.1

## Highlights

- Unified firmware install experience: manual `.bin` upload now uses the same overlay-based flow as the GitHub update path, avoiding the bare "FIRMWARE UPDATED" page in iOS captive Wi-Fi browsers.
- Software Update section restructured: current version shown below the title, automatic/manual controls hidden when Automatic Updates is set to On.
- GitHub update reliability improvements: dedicated timeout constants, connect timeout added, and 5xx server errors now produce a clear user-facing message instead of a generic failure.

## Fixes

- **Manual firmware install overlay** - uploading a `.bin` file now stays on the setup page and shows the same install progress overlay as an online update. The bare confirmation page that appeared in iOS captive network browsers is gone.
- **Auto-update toggle visibility** - switching Automatic Updates to On now immediately hides the manual update controls (Check for Update, file picker, Install button) in both the main script and the iOS fallback controller.
- **Install button enabled on file select** - the Install button is now correctly enabled when a `.bin` file is chosen, including in the iOS fallback path where the main script may not have loaded yet.
- **Software Update card gap** - the Software Update card now has the same spacing as all other settings cards on the page.
- **GitHub Check for Update timeout** - the GitHub release check now uses its own 30-second timeout constant (`GITHUB_HTTP_TIMEOUT_MS`) and sets both read and connect timeouts, fixing the 504 errors seen when the GitHub API was slow to respond.
- **GitHub 5xx errors** - GitHub server errors (500–599) now show a clear "temporarily unavailable" message rather than a generic failure code.
- **Setup page load speed** - the HTML string pre-allocation was increased from 72 KB to 90 KB, eliminating a heap reallocation that was causing a 2–3 second delay on first page load.

## Builder Notes

```bash
./tools/FreedomClockSecurityTool.command build-manual-update --release-name freedom-clock-v2026.05.23.1 --board e213
./tools/FreedomClockSecurityTool.command build-manual-update --release-name freedom-clock-v2026.05.23.1 --board e290
```

Use the `.bin` file matching the physical board model. Do not flash an E213 build onto E290 hardware, or the other way around.
